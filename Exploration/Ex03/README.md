# PRT(Peer Review Template)
- [ ]  **1. 주어진 문제를 해결하는 완성된 코드가 제출되었나요?**
    - 문제에서 요구하는 최종 결과물대로 첨부하였음
    - 회고와 실험개요들을 통하여 실험 조건과 결과들을 분석한 부분이 상세하여 본 코드들을 설계한 내용을 쉽게 파악할 수 있음
 
- [ ]  **2. 전체 코드에서 가장 핵심적이거나 가장 복잡하고 이해하기 어려운 부분에 작성된 
주석 또는 doc string을 보고 해당 코드가 잘 이해되었나요?**
    - 해당 코드의 기능, 존재 이유, 작동 원리 등을 주석을 통하여 자세히 기술하였고 잘 이해하고 있음
        
- [ ]  **3. 에러가 난 부분을 디버깅하여 문제를 해결한 기록을 남겼거나
새로운 시도 또는 추가 실험을 수행해봤나요?**
    - 문제 원인 및 해결 과정을 잘 기록하였고 본인만의 시도와 실험 과정이 잘 기록되어 있음
        
- [ ]  **4. 회고를 잘 작성했나요?**
    - Residual 모델과 PlainNet모델에 대해서 실험을 통해 주관적인 결과가 회고에 잘 드러나있음 
    - "입력 데이터를 출력에 더해주는 간단한 아이디어(Residual Learning)가 기울기 소실을 막고 정보 전달을 원활하게 하여, 50층 이상의 초심층 네트워크도 학습 가능하게 만듦"이라는 문구에서 Skip Connection의 기능을 잘 이해하고 있음
        
- [ ]  **5. 코드가 간결하고 효율적인가요?**
    - 코드의 중복을 최소화하여 복잡한 함수들을 간략화하는데 노력하였음이 보였음
    - ResNet과 PlainNet 모델의 비교들을 한 번에 알 수 있는 그래프를 작성하였으면 더 품질이 높을 것으로 판단됨
